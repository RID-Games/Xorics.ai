# Xorics Android app

Native Android client for the Xorics bridge. **Rung 1: prove-it build** — a single
button that does one voice roundtrip against the bridge over Tailscale:

    record (6s) -> /stt -> /v1/chat/completions -> /tts -> play

It shows the transcript and Xorics's reply on screen. This is the Android equivalent
of the Termux probe's screen-on roundtrip: the goal is to confirm the project builds,
installs, reaches the bridge, and completes a clean roundtrip — **before** layering on
the background service, VAD, and wake word.

## Build it (laptop, Android Studio)

1. Commit this folder to git on RIDGames and pull it on the laptop.
2. Open the folder in **Android Studio** (File -> Open -> select `xorics-app`).
3. Let Gradle sync. Two things may happen on first open:
   - **"Gradle wrapper not found" / missing `gradle-wrapper.jar`.** This binary isn't
     in the repo (it can't be shipped as text). Android Studio will offer to generate
     it, or run in the project root:  `gradle wrapper --gradle-version 8.10.2`
     (Android Studio bundles Gradle, so its Terminal tab has `gradle` available).
   - **A prompt to upgrade AGP / Gradle.** Accept Android Studio's suggestion if it
     offers one — it auto-migrates the version numbers. The pinned versions
     (AGP 8.7.3, Kotlin 2.0.21, Gradle 8.10.2, compileSdk 35) are known-good, but
     newer is fine.
4. Plug in the phone (USB debugging on), pick it as the target, and Run.

`local.properties` (pointing at the laptop's SDK) is generated by Android Studio on
first open and is gitignored — you don't create it.

## Before it can reach the bridge

- **Tailscale must be up on the phone** (per-app VPN is fine — the app only talks to
  the bridge). The app uses `https://ridgames.tail893cf4.ts.net`.
- The bridge + whisper + Kokoro must be running on RIDGames (now systemd services).

## Where the bridge URL / token live

`app/src/main/java/com/rid/xorics/Bridge.kt`:
- `BASE`  — `https://ridgames.tail893cf4.ts.net`
- `TOKEN` — `xorics-app` (matches `XORICS_BRIDGE_TOKEN` if you set one; currently any value works)

## Structure

    app/src/main/java/com/rid/xorics/
        MainActivity.kt   button + runtime mic permission + the roundtrip
        Bridge.kt         stt / chat / tts HTTP calls (reused by the service later)
        Audio.kt          MediaRecorder record + MediaPlayer playback (reused later)

## Next rungs (after this builds and runs)

1. **Foreground service** — move the loop into a service with a persistent
   notification so it runs with the screen locked (the whole point of going native).
   `Bridge` and `Audio` are already factored to drop straight in.
2. **VAD** — auto-detect end of speech instead of a fixed 6s window.
3. **Wake word** + **speaker-ID** — only respond to "Hey Xorics" in your voice.

## Wake word (v0.6, rung 3a) — Vosk on-device detection

This is a PROOF: Vosk runs fully on-device and only flashes/vibrates when it hears the
wake word. It is NOT wired to the bridge yet. It proves three things before we build on
it: the dependency + native libs build, detection works, and it leaves audio alone.

### One-time model setup (required — the model is NOT in the APK)
1. Download `vosk-model-small-en-us-0.15.zip` from https://alphacephei.com/vosk/models
   (~40 MB) and unzip it. You get a folder containing `am/ conf/ graph/ ivector/`.
2. Rename that folder to `model`.
3. Build + install the app once (so its data dir exists), then with the phone on USB:
   `adb push model /sdcard/Android/data/com.rid.xorics/files/`
   You should end up with `.../files/model/am/`, `.../files/model/conf/`, etc.

### Test
- Tap **Wake Word Test**. The notification shows "Loading model…", then
  "Listening for wake word…".
- Speak. The notification shows the LIVE transcript ("… <words>") — this is how we see
  what the small model emits, including what it turns "Xorics" into.
- Say the wake word (ZOR-ics or SOAR-ics). On a match it vibrates and shows "✓ WAKE: …".
- Stop with the notification Stop, the in-app Stop Wake, or Force Stop.

### Tuning
"Xorics" is not a dictionary word, so Vosk approximates it. WAKE_VARIANTS in
WakeService.kt is a starter list — watch the live transcript, note what the word becomes,
and we extend the list (or switch to a grammar) from real output.

### Known risks (this is the least-tested build)
- If sync fails on the Vosk dependency, try version 0.3.70 or 0.3.47.
- If JNA errors at build, add: implementation("net.java.dev.jna:jna:5.13.0@aar")
- "Model missing" notification = the adb push didn't land in .../files/model.
